# 🏦 Banking Analytics Project

## 📌 Project Overview
This project demonstrates a complete **end-to-end Data Analytics pipeline** built entirely through self-learning (no courses or tutorials).

The workflow starts from raw banking data and goes through cleaning, analysis, and visualization to generate meaningful business insights.

---

## 🚀 Workflow

Raw Data (CSV)  
↓  
Data Cleaning (Python - Pandas)  
↓  
Processed Dataset (banking_dataset_final.csv)  
↓  
SQL Server (Data Storage + Business Queries)  
↓  
Power BI Dashboard  

---

## 🛠️ Tech Stack

- **Python** (Pandas, NumPy) – Data Cleaning & Transformation  
- **SQL Server** – Data Storage & Querying  
- **Power BI** – Dashboard & Visualization  
- **CSV Dataset** – Raw Data Source  


---

## 📊 Dashboard Features

### 🔹 Key Metrics
- Total Transactions  
- Total Customers  
- Average Balance  
- Average Transaction  
- Maximum Transaction  

### 🔹 Analysis Performed
- Customer distribution by branch  
- Transactions by category (Food, Bills, Salary, etc.)  
- Monthly transaction trends  
- Transactions by weekday  
- Cash flow (Inflow vs Outflow)  
- Credit vs Debit comparison  

---

## 📈 Key Insights

- Debit transactions are higher than credit transactions  
- Spending is highest in categories like **Food and Shopping**  
- Certain days show higher transaction activity  
- Cash outflow is greater than inflow  
- Some branches have significantly more customers  

---

## 🔍 SQL Analysis

SQL Server was used to answer business questions such as:

- Customer count per branch  
- Monthly transaction trends
